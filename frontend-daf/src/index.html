    Fast parallel sorting and other math functions

    kbowers-jump merged to firedancer-io/firedancer at 2025-05-11 04:34:17

    Fast parallel sorting and other math functions By popular demand, this is the last parallel sorting algorithm to install. This is also cache and NUMA friendly. It has the best asymptotic behavior (average is roughly O((N lg N)/T) + O(lg T)). But it requires FD_HAS_ALLOCA, scratch memory (even if not running a stable sort) and uses a more stack memory on the main thread than the other parallel algorithm implementations. Under the hood, it randomly samples the input array to find reasonable pivots that split the array into T approximately uniform partitions. Then it does a coarse parallel counting sort to organize the array into those partitions. Then it embarassingly parallel sorts the T partitions on T threads. Also included in this are various functions to compute integer square roots and integer cube roots. These work anyway deterministically with reasonably fast O(1) behavior and are exact (even in situations where a double precision libm would have round error). Originally thought was going to use these to optimize the thread count (see the notes about optimizing the thread count) and wrote them while stuck diverted to Allentown for a few hours in the Newark airport ground stop a week or so ago but ultimately did not. Since they are done and tested, might as well not let them go to waste.
